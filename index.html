Day 1 · 地基

Claude Code 认知 + 环境搭建 + 数据层

Claude Code 心智模型 · FastAPI 项目骨架 · 新浪行情解析 · VWAP 计算 · WebSocket 推送
Python FastAPI WebSocket VWAP
▶
🧭 模块 1-1：Claude Code 心智模型 60 min
Claude Code 不是代码补全工具，而是能理解业务逻辑、做架构决策的 AI 工程师。学会写高质量需求比学会写代码更重要。
Claude Code vs Copilot vs ChatGPT：定位与场景的本质区别
高质量 Prompt 三要素：给背景、给约束、给预期格式
遇到报错的标准流程：截图/粘贴错误 → 让 Claude 诊断 → 最小化修改
演示：5分钟让 Claude 读懂一个300行 Python 文件并找出潜在 bug
🤖 典型提示词示例
"读取 app.py，告诉我整体架构、数据流向，以及你认为最脆弱的3个地方，用中文。"
⚙️ 模块 1-2：开发环境搭建 45 min
全程用 Claude Code 辅助配置，遇到依赖冲突和 SSL 报错直接甩给它处理。
Python 3.11 + FastAPI + uvicorn + akshare 环境安装
项目目录规范：modules / static / data 分层设计
start.sh 一键启动脚本的基础版本
📡 模块 1-3：A 股实时行情接入 + VWAP 计算 90 min
解析新浪行情 API 每一个字段，掌握 VWAP 的正确计算方式，搭好双数据源容灾。
新浪行情字段完整解析：price / prev_close / open / amount / volume
VWAP 正确公式：amount ÷ (volume × 100)，单位换算是常见踩坑点
主源（新浪）+ 备用源（腾讯）自动切换逻辑
字段安全原则：前后端字段名必须完全一致，pre_close ≠ prev_close
🤖 典型提示词示例
"新浪行情返回逗号分隔数据，帮我写解析函数，计算 VWAP 加入返回结果，加好异常处理。"
🔄 模块 1-4：WebSocket 实时推送架构 75 min
实现服务端主动推送，前端订阅即刻收到行情，断线3秒自动重连。
FastAPI WebSocket 路由 + 客户端订阅协议设计
前端 onmessage 接收并合并到 state.quotes
_lastQuoteTs 时效监控：交易时段超3分钟未收到数据触发警告
数据健康四类异常：WS断线 / 行情过期 / VWAP=0 / 价格=0

Day 2 · 大脑

技术指标 + 结构相位 + 评分模型

P1/P2/P3 结构相位 · 多维评分模型 · 大盘联动动态阈值 · Qwen AI 解读接入
P1/P2/P3 评分模型 Qwen AI 大盘环境
▶
📐 模块 2-1：结构相位分析 P1 / P2 / P3 90 min
A 股短线最重要的买入框架：三个相位共振时信号强度远超单一指标。
P1（趋势完好）：MA5上行 + 近期不破前低，趋势是地基
P2（回调到位）：从高点回落 3-8%，缩量，位置是关键
P3（当日企稳）：缩量收阳 / 低开高走，时机是触发器
用 Claude Code 将自然语言规则直接转为 Python 判断函数
为什么 P1+P3 是最低门槛，P1+P2+P3 才是高质量信号
🤖 典型提示词示例
"实现 P2 判断：从近20日最高点回落超3%但不超8%，且近5日均量 < 20日均量×0.7，返回 bool 和置信度。"
🧮 模块 2-2：多维评分体系 + 大盘联动 90 min
将多个指标整合成 0-100 评分，并让评分门槛随大盘状态动态调整。
评分维度：趋势 / 量能 / 价格位置 / 大盘 / 板块共振
baseScore（纯技术分）vs score（风控扣分后综合分）的设计意图
大盘环境三档：strong / normal / weak，驱动全系统风险偏好
动态阈值：正常/强势→65分入场，弱势→72分，防止熊市抄底
风控拦截 riskBlock：跌破关键支撑一票否决，不管评分多高
🤖 模块 2-3：接入 Qwen 生成 AI 解读 60 min
让 AI 对量化信号做自然语言解读，生成交易者能直接看懂的操作建议。
DashScope API 接入（兼容 OpenAI 格式，改一行 base_url 即可）
System Prompt 设计：角色定位 + 字数限制 + 禁止废话
将结构化评分结果注入 Prompt，让模型结合数据解读而非空谈
流式输出（streaming）实现打字机效果，提升使用体验

集成专业级图表，展示 VWAP 包络线、成交量、买卖信号标注，WebSocket 实时驱动刷新。

Day 4 · 神经

提醒引擎 + 做T信号 + 风控体系

买入/加仓/做T 三套逻辑 · 冷却机制 · Toast 架构 · 数据异常拦截 · 持仓异动告警
买入提醒 做T信号 风控拦截 节流防抖
▶
🟢 模块 4-1：买入 / 加仓提醒逻辑 75 min
核心原则：宁可漏报，不能误报。误导买入比没有系统更危险。
_evalBuyCondition：相位 + 评分 + 价格区间三重门槛
低分入场（55-64分）强制要求 P1+P2+P3 全满足
加仓防飞刀：score < 50 时屏蔽所有加仓，避免趋势破坏后补仓
30分钟冷却：同一只股不重复打扰，_buyAlertCooldown 实现
北京时间强制转换：避免时区错误导致交易时段判断失效
🤖 典型提示词示例
"_evalBuyCondition 现在55分的股票也触发了，帮我分析根因并修复，保持P1+P3条件不变，修改范围最小化。"
⚡ 模块 4-2：超短线做T信号引擎 90 min
2-4天持股周期的做T逻辑：目标是锁利润和防守撤退，而非降成本。
三种信号：急拉高抛（橙）/ 破位减仓（红）/ 均线低吸（紫）
急拉判断用 VWAP 正乖离 >3.5% 而非"开盘速度代理"（防慢牛误判）
早盘时间锁：9:30-10:00 VWAP 失真，屏蔽破位/低吸信号
硬性止损底线：日内跌幅 <-3% 即使无 VWAP 也触发破位减仓
15分钟冷却（日内节奏比买入提醒更快）
🔔 模块 4-3：提醒 UI + 推送性能优化 60 min
让提醒既不打扰也不遗漏，行情高频推送时保持页面流畅不卡顿。
抽屉面板设计：买入/加仓 与 做T 两个 Tab 视觉完全隔离
Toast 容器：Flex column-reverse 解决动态 bottom 计算导致的重叠错位
1.5秒节流：WebSocket 高频推送时每1.5秒最多触发一次检查
触发时机三路：定时5分钟 / WS推送节流 / 单股信号加载完成后
🛡️ 模块 4-4：数据健康监控 + 风控拦截 45 min
数据不对比没有信号更危险，有问题必须第一时间点名告诉用户哪只股出了什么问题。
_checkDataHealth：四类异常检测，精确到具体股票名称
面板红色警告条：有问题显示，数据恢复自动消失
price=0 / vwap=0：跳过计算，拒绝产生基于错误数据的信号
尾盘异动监控：14:44后量比≥1.5x 且跌破 VWAP 触发持仓告警

Day 5 · 验证 & 上线

回测验证 + 本地部署 + 持续迭代

历史回测框架 · 信号胜率统计 · 参数校准 · 本地自启动部署 · AI 协作迭代方法论
回测框架 胜率统计 自动部署 迭代方法论
▶
📊 模块 5-1：历史数据回测框架 90 min
用历史 K 线数据模拟信号触发，统计每类信号的真实胜率和风险收益比。
akshare 获取历史日 K 线数据，构建回测数据集
历史数据回放：逐日模拟 P1/P2/P3 判断 + 评分计算
信号胜率统计：触发后3天/5天涨幅分布，识别哪些参数真的有效
最大回撤计算：验证风控拦截是否真的减少了亏损
用 Claude Code 自动生成回测报告和可视化图表
🤖 典型提示词示例
"帮我写回测脚本，对某只股票过去1年的数据运行 _evalBuyCondition 逻辑，统计触发后5日平均涨幅和胜率。"
🔬 模块 5-2：参数校准与信号优化 60 min
回测结果告诉你哪些阈值设对了、哪些需要调整，用数据说话而不是感觉。
评分阈值敏感性分析：60分/65分/70分的胜率差异
P2 回调幅度区间校准：3-8% 是否是最优范围
大盘环境分层验证：强市/弱市的信号质量差异有多大
做T信号回测：VWAP 偏离 3.5% 阈值的历史准确率
过拟合警示：回测好看不代表实盘好，如何避免过度优化
🚀 模块 5-3：本地自动部署 45 min
让系统每天开盘前自动启动，开盘后立即开始监控，完全不需要手动操作。
start.sh 完整版：端口检测 + 旧进程清理 + 日志轮转
macOS launchd 定时任务：工作日 9:25 自动启动
数据库定期清理：防止分时缓存无限增长
后续扩展方向：企业微信/钉钉推送 / 云端部署 / 移动端适配
🔄 模块 5-4：用 Claude Code 持续迭代的方法论 45 min
这门课最重要的内容：系统不是一次写完的，是一次次实盘误报之后逐步校准的。
收到优化建议 → 评估可行性 → 让 Claude Code 实现 → 回测验证
遇到实盘误报 → 截图给 Claude → 分析根因 → 最小化改动修复
严谨性原则：查清楚数据再动手，不要在错误假设上叠加逻辑
AI 协作的边界：哪些决策必须人来做（止损纪律），哪些交给系统
量化工具辅助决策，不替代判断——系统给信号，你来按不按
🤖 典型提示词示例
"昨天系统对某只股触发了买入提醒但它随后跌了，帮我分析是哪个判断条件失效了，以及怎么避免下次误报。"

贯穿全程的核心理念
⚡
AI 加速，人来决策

安全是底线
